    Functions

    Functions are chunks of code (aka blocks of code), that when triggered, are a set of instructions regarding what the computer program should do. Think about our flashlight example: the function you saw turned the light on or off. 

    Let's look deeper at the syntax of a function:

    function nameOfFunction() {        
    // code to be executed
}

    A function is a set of instructions, just waiting to be triggered. However, functions are like actors in the wings of a play: they aren't always on stage repeating their lines; they have to wait to be called on. Another metaphor is that functions are like a recipe.  Once we have the recipe for chocolate chip cookies we know how to make cookies, but that doesn't mean we immediately have cookies once we get the recipe.  We have to actually call the actor on stage or work through the steps and make the cookies.

    Calling or Invoking a Function

    So, the above example of a function is the recipe card, or the actor's lines. To actually get the function to work, we have to call it, and we do that by calling it by its name and adding ()

    nameOfFunction();

    Here's an example:

    function counter() {    
    for (var num = 0; num <= 5; num++) {        
        console.log(num);    
    }
}
counter();    // run the function
counter();    // run the function again

    Since the function counter() is written, we can call on it as many times as we'd like! 

    Note: You might be wondering what all that stuff in between the { and } does. Well, good news: you can learn that by completing the Algorithm App's Lessons 1 - 4. 

    Functions are powerful and grow in complexity. Throughout the bootcamp, you will learn a lot more about functions.
