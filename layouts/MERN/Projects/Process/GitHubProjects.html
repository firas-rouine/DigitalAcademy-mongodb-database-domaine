    GitHub Projects

    Learning Objectives

    • Create a board and table using GitHub Projects

    Keeping your project goals organized is the best way to ensure you are able to meet your MVP on time. You should use GitHub's built-in project-tracking features to keep everything you need in one place.

    If you have not already done so, sign up for a GitHub account. GitHub is an excellent tool to post your projects and collaborate with teammates on larger projects.

    This module details the steps necessary to create your project repo and set up everything you need to start tracking your project progress. This tutorial does not cover any steps related to pushing code to the repository.

    GitHub Projects Tutorial

    To begin tracking your project, you must first make a repository in GitHub. At this time you do not have to have any code pushed into the project. You only need a project to exist.

    Once your repository is created, you can access the Projects tab. Select it to move over to the Projects tab window.

    From within the Projects tab, locate this button on the right to link a project. Select the dropdown, change it to "New Project", and click the button to open the New Projects window.

    From here, change the selection on the left bar to "Board", give the project a name, and press the Create button.

    This will generate your new Board.

    This will create 3 columns: Todo, In Progress, and Done. You can add items to any of the columns by selecting "Add Item" at the bottom of the appropriate column.

    Once the item is added, you can click on it to add further details by clicking the Edit button. If you want to take your descriptions to the next level, you can use Markdown to make your notes clear and organized.

    Each item can also be dragged and dropped between the columns as you complete or progress on your items.

    If you would like to add another column, you can do so by selecting the plus (+) icon to the far right of the columns.

    Next, we recommend that you also add a Table to your project. A table will be another way to visualize your work and will allow you to add more details to each item on your board. To create a table, select the "New View" button and select "Table".

    This will automatically generate a table with any items created on your board.

    From here, we recommend that you add two new columns: one to specify a due date and another to specify a priority. To create extra columns in a table, select the plus (+) sign on the far right of the header of the table.

    You can now update each item with its relevant information.

    Update or add any extra features as needed to create an organization tool that works for you. Update the name of the project and the tab names, and assign collaborators to certain items on the board.

    Adding Assignees can be a great way to delegate tasks and split up work between teammates. If you are working with others on a project, make sure the owner of the GitHub repo adds everyone as a collaborator so they can be part of the project.

    The final optional step is to rename each tab so it is easy to navigate.

    With that done, you have your project board all set up and ready to use! In the next assignment, you will create your Project board for your upcoming project.
