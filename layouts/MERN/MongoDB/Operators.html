    Operators

    Objectives

    • Introduce using operators to modify updates and queries

    Update Operators

    Update Operators are tools to modify update queries to do specific different types of updates. MongoDB needs to be told explicitly that we're trying to modify an array, for example, otherwise we run the risk of overwititng data and losing information!

    Generally, the need for Update Operators is tied very closely to what it is you want to update. Updating fields in an object is pretty straight forward, but pushing and popping from an array can be a little bit more tricky. Ask yourself what the operation your trying to complete is, then it will be easy to find the Update Operator that applies to your situation!

    Below are some common update operators and their functions, and you can find a more exhaustive list here.

    $currentDate

    $currentDate sets the value of a field to current date as a Timestamp.

    db.ninjas.updateOne({_id: 1}, {$currentDate: {createdAt: true}})

    $inc

    $inc increments the value of the field by the specified amount.

    db.ninjas.updateOne({_id: 1}, {$inc: {likes: 1})

    $set

    $set sets the value of a field in a document. Adds the field if it does not already exist.

    db.ninjas.updateOne({_id: 1}, {$set: {favorite_color: "orange"})

    $push

    $push adds an item to an array.

    db.ninjas.updateOne({_id: 1}, {$push: {fruit: "apples"}})

    $pull

    $pull removes all array elements that match a specified query.

    db.ninjas.updateOne({_id: 1}, {$pull: {animals: "elephants"})

    $addToSet

    $addToSet adds elements to an array only if they do not already exist in the set.

    db.ninjas.updateOne({_id: 1}, {$addToSet: {weekdays: "Sunday"})

    Query and Projection Operators

    Using Query Operators, we can modify our database queries to capture more specific sets of data. Just as Update Operators, we are telling MongoDB that we want to modify what it is w are querying for. So far for many of our examples, you should notice we are querying by _id. In most cases, this is the best way to single out specific documents in your database.

    However, there are still situations where we might want multiple documents, or situations where we don't have access to any ids we could use to query. These queries could be slower and more computationally taxing for MongoDB to use, so in a large data set it might be better to find alternative solutions!

    $gt

    $gt, or greater than, is used for finding documents where the value of a field is greater than a passed value.

    db.ninjas.find({likes: {$gt: 10}})

    $not

    $not inverts a query expression and returns documents that do not match.

    db.ninjas.find({name: {$not: "Steve"}})

    We can also combine $not with other operators.

    $exists

    $exists finds all documents that include a specific field.

    db.ninjas.find({admin: {$exists: true}})

    Many of these can be combined with other operators to create even more specific queries! You can find more update operators here.
