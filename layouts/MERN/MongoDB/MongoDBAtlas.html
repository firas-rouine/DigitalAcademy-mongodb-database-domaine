    MongoDB Atlas Setup

    Objectives

    • Create an account with MongoDB Atlas and configure a data cluster
    • Install mongosh, the MongoDB Shell
    • Connect to Atlas with mongosh

    Why Atlas?

    MongoDB Atlas is a cloud computing data host that we will use to access and configure our databases. While we can install MongoDB locally, using it in the cloud will better familiarize you with how Mongo is used in actual enterprise software development. As a cross-platform service, if you're using PC, Mac, or Linux, your experience with Atlas with be largely the same.

    Setting Up Your Cluster

    1. Create a new account with MongoDB Atlas here. We recommend using a Gmail or Github account to sign in.

    2. Next we want to create and deploy a new cloud database. When doing this we always want to create a Shared Cluster, which is the free tier of Atlas.

    3. Choose a cloud provider. Pick a location close to you, but it ultimately should not matter who the provider is.

    4. Cluster tier, additional settings, and the cluster name are not important, just make sure you pick free tier.

    5. Set up a username and password to connect to your cloud hosted database. You don't need to make this too serious, we recommend a simple password.

    6. Set up IP Access by clicking Add My Current IP Address. This will restrict what networks can connect to your hosted database and make changes. If you move from location to location and use many different wifi networks, you will have to go in here and add the IP address of the network as needed. If you add the permission 0.0.0.0/0 to your IP list, any IP address can connect to your database as an admin. If you allow any IP, make sure your username and password are strong!

    Now you can launch your cluster and let it finish configuring on it's own!

    Connecting with mongosh

    Using Mongo Shell, or mongosh will allow us to connect to our database and send it commands. We will need to install the shell to use it.

    1. First, click the Connect button. We want to connect with the Mongo Shell.

    2. If we do not have the MongoDB Shell installed at this point, now is the time to do it! Select your operating system and follow the steps. If you already have the MongoDB Shell installed, skip ahead.

    Homebrew Install (Optional)

    Homebrew is a popular packager manager for MacOS. It must be installed and added to your system's path before installing the mongo shell. We'll quickly walk through the installation and add it to the system path together. We will demonstrate adding the path variable on Windows in the next section of this module. Skip there now if you're a Windows user.

    First, click the provided Homebrew link as seen in the image below:


    Note: Do not enter the brew install mongosh line yet. We will do that later.

    Next, copy the line provided on the Homebrew page and run it in your terminal. This will install the Homebrew package manager to your system. You can now use it to install packages.


    It's important to add Homebrew to your system's path. To do so, type the following into the terminal:

    echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.bash_profile

    Finally, we can install the mongo shell using homebrew. It's wise first to close your terminal and re-open it before typing the following line:

    brew install mongosh


    Path Variable (Windows Only)

    Windows users must update their $PATH variable to allow mongosh to work from Command Prompt. Let's review the steps:

    a. Download mongosh and unzip the file somewhere safe. Remember the path to this file.

    b. Press Windows + S and bring up the Search. Search for env and select edit the system environment variables. Select Environmental Variables...

    c. In System variables, Edit... the Path variable. Add the bin file path to where you unzipped mongosh.

    Using mongosh

    1. Now with mongosh installed, let's connected to our database cluster! When hitting the Connect button and following the dialogues, we eventually get to our connection string. Copy this to your clipboard.

    2. Open terminal / command prompt and paste your connection string. Hit enter, and you will be prompted for the password you set up earlier. Remember this doesn't need to be a hyper secure password, do something you can type easily!

    Your view will look slightly different depending on if you're using gitbash, terminal, or command prompt.

    You should now be connected to your remote database!
