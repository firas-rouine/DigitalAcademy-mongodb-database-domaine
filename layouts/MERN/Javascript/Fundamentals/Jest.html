    Unit Testing With Jest (Optional)

    Properly tested code is crucial to smoothly-working applications. The process of adding features and maintaining an application can very easily introduce subtle bugs. In order to ferret out these bugs before they hit production code, we can write tests. The process of TDD, or test-driven development, goes something like this:

    1. Write failing tests for the relevant functionality.
    2. Build out a working implementation of the needed functionality.
    3. Refactor your code as needed to ensure your tests pass.

    Note that you write the tests before the actual code. There are different types of tests, such as unit, integration, end-to-end, and acceptance. We're going to explore unit testing in Node, both with the built-in assertion library and a robust testing framework (Jest). When we do assertion testing, our tests determine whether our functions return the correct output when given certain inputs. Here's how we can require the native assertion library:

    const assert = require('assert');

    Say, for instance we have the following add function. Here's how we could test it.

    function add(num1, num2) {
    return num1 + num2;
}
    
assert.strictEqual(add(2, 2), 4);
assert.strictEqual(add(4, 6), 10);

    The structure is simplified, as your tests would typically be in a different file than the actual functions. What happens if the two values passed to the strictEqual function don't match? The program will exit with an error. Although this works fine for small scenarios in which there are a few functions to test, you will likely choose a more full-featured framework when working on a larger application. There are a number of such JavaScript frameworks; we're going to focus on Jest.

    To add Jest to your npm project as a dev dependency, run the following: npm i --save-dev jest. A dev, or development, dependency is one which is used only in the app development process. Once Jest is installed, you can add a testing script to your package.json file, which will run your tests. You can also optionally add a script which will watch for any file changes:

    ...
    "scripts": {
        "test": "jest",
        "test:watch": "jest --watch"
    }
    ...

    To run your script, type npm run test in the terminal. Now, let's see how we can test our add function using Jest. Assume we have a funcs.js file in the same directory which exports the add function. Here's how our file might look:

    // add.spec.js
const { add } = require('./funcs');
    
describe('add', () => {
    test('correctly returns the sum of two numbers', () => {
        expect(add(2, 2)).toBe(4);
        expect(add(4, 6)).toBe(10); 
    });
});

    When your tests are all passing, you'll see all green in your terminal. Conversely, when one or more tests is not passing, you'll be given the context along with the expected and received output, which you will use in the refactoring phase to fix your code. Notice how the syntax closely matches the way we think about our functions: "describe the add function; test that it correctly returns the sum of two numbers; expect add(2, 2) to be 4; expect add(4, 6) to be 10." This was a small, contrived example, but hopefully you can see how handy this will be in larger applications. We can safely refactor functions knowing that our automated tests will catch us if the refactoring causes unexpected output changes.
