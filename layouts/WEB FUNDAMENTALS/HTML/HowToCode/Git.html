    Git

    One of the most important tools to get familiar with is git--the most commonly used program for version control. Git will allow us to keep track of changes we make to our code, let us make versions that we can use to test new features and merge them in if they work or go back to an old version if it was a mistake. Git takes time to master so we'll introduce the basics now so we can start using it on all of our projects.

    Installation & Set-Up

    To get started, we need to install Git on our computer.

    Visit this link and follow the directions for your operating system: http://git-scm.com/book/en/v2/Getting-Started-Installing-Git

    To verify if your installation worked, open your terminal (Or Git Bash) and type git at the command line. If you see a bunch of options and list of git commands, your installation was successful!

    The first thing you should do after installing Git is to set your username and e-mail address. This is important because every Git commit uses this information.

    Enter your shell (Terminal or Git Bash) and enter the following:

    git config --global user.name "John Doe"
    git config --global user.email johndoe@example.com

    You can verify that Git stored your settings by passing git config the --list parameter: 

    git config --global --list

    These are the only two required configuration values, but Git allows you to configure more than 130 different settings. Most of these youâ€™ll never need to tweak, but there is one extra change that is useful if you like color-coded output.

    Repositories

    The starting point for each project that git manages is initializing a `repository`. When we look at the icons on the left of VS Code the branching icon represents `git`. Inside there we can `Initialize a Repository`. This is a hidden folder that `git` will manage for us that keeps track of all of our files and the changes we make to them.

    Adding Changes

    We can add all of the changes to the repository by clicking on the plus icon next to all of the changes or the one at the top that says `Staged Changes`. We can then `commit` these changes by writing a message. Something like "initial commit" would be good for now but later we'll want these messages to indicate what changes we've made. Then click on the check to make the `commit`.

    Remote Repositories - GitHub

    While our code is being tracked locally on our computer, one of the more important reasons to use git is that it makes backing up our code online easily using sites like GitHub, GitLab, BitBucket, etc. GitHub is both popular and free. We will be making an account on GitHub.
