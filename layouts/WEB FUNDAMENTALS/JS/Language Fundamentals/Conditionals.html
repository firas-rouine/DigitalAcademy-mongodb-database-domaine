    Conditionals

    Objectives

    • Learn about Booleans
    • Learn about Comparison Operators
    • Learn about if, else if, and else
    • Learn about Logical And (&&) and Logical Or (||)

    Booleans and Comparison Operators

    By now we've seen a lot about Number and String, but haven't talked much about Boolean.

    Boolean is another data type and it can only have 2 possible values, true and false. We can declare them as variables

    var isSunny = true;
var isRaining = false;

    And there are a number of operators that will return a Boolean.

    var number = 5;
console.log(number < 3); // this will log `false`

    Operators that return a Boolean
    Operator Explanation
    a == b
    		 true if a is equal to b
    a != b true if a is not equal to b
    a < b true if a is less than b
    a > b true if a is greater than b
    a <= b true if a is less than or equal to b
    a >= b true if a is greater than or equal to b

    Conditionals

    Conditonals provide us a way for our code to make a decision, to branch off in one direction or another.

    var isSunny = false;
var isRaining = true;
    
if(isSunny) {
    console.log("Wear sunscreen");
}
    
if(isRaining) {
    console.log("Bring an umbrella");
}

    In the above code we will bring an umbrella because isRaining is true.

    Note: new programmers often write conditionals that look like the following code

    if(isSunny == true) {
    console.log("Also wear a hat");
}

    There is nothing wrong with code like this, but if the variable isSunny is already going to be true or false it's not necessary to use the comparison operator with true. However if this example is easier for us to read we should feel free to write it this way. Being a good programmer is about writing code that we understand not writing the most complicated code we can get to work.

    else and else if

    Sometimes we want to have an alternative to the code if the condition isn't true, in this example if the current day isn't Monday it will let us know that "Today is alright!"

    Note: the .getDay() part of this code will return a number between 0 and 6 where 0 is Sunday and 6 is Saturday

    var today = new Date();
if(today.getDay() == 1) {
    console.log("I hate Mondays!");
}
    
if(today.getDay() != 1) {
    console.log("Today is alright!");
}

    This code block can be rewritten using an else to avoid having to write a second condition

    var today = new Date();
if(today.getDay() == 1) {
    console.log("I hate Mondays!");
} else {
    console.log("Today is alright!");
}

    The else part of the code will run whenever the current day isn't Monday (.getDay() of 1).

    We can mix in additional cases with the else if condition

    var today = new Date();
if(today.getDay() == 1) {
    console.log("I hate Mondays!");
} else if(today.getDay() == 5) {
    console.log("Friday? More like Fri-yay!");
} else {
    console.log("Today is alright!");
}

    Generally it's a good idea to write the most specific condition first, followed by less specific conditions, followed by an else to handle cases where no previous condition matched up.

    Multiple conditions

    If we want to pick a good day to go for a walk, we probably want to find a time when it's (1) not too cold and (2) it's not raining. We can nest conditions inside of each other and the message inside will only be logged if both conditions are true (temperature is greater than 50 and it's not raining).

    var temperature = 60; // let's assume this is degrees Fahrenheit
var isRaining = false;
    
if(temperature >= 50) {
    if(!isRaining) {
        console.log("This is a good day to go for a walk!");
    }
}

    This can be written in a simpler way by using some additional operators

    var temperature = 60; // let's assume this is degrees Fahrenheit
var isRaining = false;
    
if(temperature >= 50 && !isRaining) {
    console.log("This is a good day to go for a walk!");
}

    The logical and (&&) will only be true if the values on both sides are both true. If we wanted to check if either of the values are true, we can use the logical or (||) operator instead.

    Logical And and Logical Or
    Operator Explanation
    a && b true if both a and b are true
    a || b true if either a or b are true

    Note that || isn't an exclusive "or" (xor) as it still returns true if both a and b are true

    Modulo Operator

    Sometimes we need to determine if a number is even or odd, in code this is often accomplished using the % or modulo operator.

    var is5even = 5 % 2 == 0;
var is500even = 500 % 2 == 0;
    
console.log(is5Even);   // false
console.log(is500Even); // true

    Modulo works by determining the remainder, in the case of an even number we can divide it through by 2 and have no remainder (or decimal). If instead we were checking a number like 5, we can divide 2 in twice and we'll have a 1 left over. This is what modulo does, it finds the remainder!

    Applying modulo to other values

    If we wanted to see if a number is divisible by 3 evenly, we can use the same approach we did to find even or odd numbers, just swapping out the 2 for a 3!

    var is78DivisibleBy3 = 78 % 3 == 0;
console.log(is78DivisibleBy3); // true
