    Java Versioning

    Learning Objective:

    • Understand Java versioning and LTS


    Prevalence in the industry. If you become a Java developer when you graduate, you are very likely to come into contact with Java 8. However, the industry is currently making the transition to Java 17, which is the most recent Long-Term Support (LTS) version. As we move into Java Spring and upgrade to Spring Boot 3, the version will become much more important. The newest version of Spring Boot requires Java 17.

    Long Term Support. There are many released versions of Java, but generally it's best to develop (and learn) in one of the stable released versions, also known as Long-Term Support (LTS) versions. While Java 8 still has the longest extended support, until 2030, its premier support ended in March of 2022.

    How it will affect your Java learning. In short, it won't. Java is a language that has been around a very long time, and because it is so powerful and effective it doesn't tend to change very much over time.

    Note about VS Code Java Extensions: We suggest refraining from using Java extensions in Visual Studio Code, as its support tends to trend toward the newer JDK versions. Most Java developers use Java-specific IDEs such as IntelliJ or Spring Tool Suite (STS), an Eclipse-based IDE which we'll learn very soon in the course.


    You can read more about versioning here: Oracle Java SE Support Roadmap
