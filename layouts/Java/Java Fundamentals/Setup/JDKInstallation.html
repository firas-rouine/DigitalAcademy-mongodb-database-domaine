    JDK Installation

    Learning Objectives:

    • Install the JDK
    • Be ready to start coding!

    The Java Development Kit is the software you'll need to run Java code. You will use the JDK to compile your java code into .class files which contain a compact version of the source code you write, called bytecode. It can also be compiled into .jar files but for now we'll only compile into .class files. After compiling, we can run the bytecode with tools that are included in the JDK.

    Installation

    Note: We are using Amazon Corretto 17 because it offers Long Term Support and is open source.

    1. To download the JDK, visit the following link and select the appropriate package according to your operating system.

      Downloads for Amazon Corretto 17

    2. After you download, run the file and follow the instructions of the installer. You should be able to click "Next" until you reach the end of the installation.

    To check if the installation has been successful, open your command prompt and type java -version. You should see the following:

    Path Error

    If you see the error above, it means that the installer did not put Java into your PATH variable automatically. This error is most common for Windows users.

    Setting Path Variables (Windows 11)

    • From the Desktop, click on the Windows icon and type in ‘path’
    • Click on ‘Edit the system environment variables’
    • Click on ‘Environment Variables’

    From there, follow these steps as shown in the graphics below.

    1. Click the Environment Variables button near the bottom of that tab.
    2. In the Environment Variables window, highlight the Path variable in the "User variables" section, or if the Path variable does not exist yet, add it.
    3. Click the Edit button.
    4. With the edit window open, listing path variables, click Browse and navigate to your c-drive Program Files. In the Amazon Corretto folder, you should see a jdk folder listed.
    5. Select the bin folder within your jdk folder.
    6. Click OK on each of the dialogue boxes to save.
    7. Open your terminal and run java -version once again.

    SE VS EE

    A common point of confusion is Java Standard Edition vs. Java Enterprise edition. What you need to know about this is that Java EE is just a standard set of libraries that allow developers to create web applications in a particular way. We are mainly developing Spring Boot applications in this course, but we will explore some of the Java EE technologies that are common to both stacks.
