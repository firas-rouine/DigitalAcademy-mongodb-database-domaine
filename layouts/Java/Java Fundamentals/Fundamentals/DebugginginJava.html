    Debugging in Java

    Learning Objective:

    • Students will use print statements to check and debug their code

    Debugging is an important skill in any language. As you become a more experienced programmer, you will find yourself using debugging tools to effectively find issues in your code. For now, we will focus on using print statements.

    With the introduction of conditionals, methods, and loops, we have moved away from code that runs line by line and introduced a new flow to our programs. With this complexity come new challenges. Let's go over the issue we encountered in the loops module one more time.

    import java.util.ArrayList;
public class Loops {
    public static void main(String[] args){
        ArrayList<String> snacks = new ArrayList<String> ();
        snacks.add("Apples");
        snacks.add("Almonds");
        snacks.add("Pretzels");
        snacks.add("Yogurt");
        System.out.println(snacks);
        for(int i=0; i<snacks.size(); i++) {
            if(snacks.get(i).charAt(0) == 'A') {
                snacks.remove(i);
            }
        }
        System.out.println(snacks);
    }
}

    If our goal is to remove all snacks starting with the letter 'A', we would get an unexpected outcome:

    [Almonds, Pretzels, Yogurt]

    This is a great time to make use of a print statement to analyze our code. Let's place the print statement inside of our loop to see what is happening as we move through our list. We will place the print statement inside the conditional statement that is meant to delete all snacks starting with the letter 'A'.

    for(int i=0; i<snacks.size(); i++) {
    if(snacks.get(i).charAt(0) == 'A') {
        System.out.println(String.format("i = %d AND we are removing %s", i, snacks.get(i))); // new line
        snacks.remove(i);
    }
}

    Our output will now look like this:

    [Apples, Almonds, Pretzels, Yogurt] // our initial snacks list
i = 0 AND we are removing Apples // our new print statement
[Almonds, Pretzels, Yogurt] // the output

    We can see that our condition of snacks starting with the letter 'A' is only met once and only 'Apples' are deleted. For some reason 'Almonds' are slipping through. Let's go a little deeper by adding another print statement. This time, we will add the print statement before we get into the if statement.

    for(int i=0; i<snacks.size(); i++) {
    System.out.println(String.format("i = %d AND snack = %s", i, snacks.get(i))); // new line
    if(snacks.get(i).charAt(0) == 'A') {
        System.out.println(String.format("i = %d AND we are removing %s", i, snacks.get(i)));
        snacks.remove(i);
    }
}

    Now we get the following output:

    [Apples, Almonds, Pretzels, Yogurt]
i = 0 AND snack = Apples // our new print statement
i = 0 AND we are removing Apples 
i = 1 AND snack = Pretzels // our new print statement
i = 2 AND snack = Yogurt // our new print statement
[Almonds, Pretzels, Yogurt]

    We have found our problem! When our loop starts, 'Apples' are at index value 0, 'Almonds' are at index value 1, 'Pretzels' are at 2, and 'Yogurt' is at 3. However, as we remove 'Apples', all other items shift down by one. This means that we never check 'Almonds' since they take on index value 0 and at that point our loop is already checking index value 1. That is why we end up with a final list of [Almonds, Pretzels, Yogurt]. Now that we know where our issue is, we can add a new line that goes back one index value each time we remove an item, thus making sure that we check every snack in our list.

    for(int i=0; i<snacks.size(); i++) {
    if(snacks.get(i).charAt(0) == 'A') {
        snacks.remove(i);
        i--; // Here we move one step back
    }
}
System.out.println(snacks);

    Finally, we get the desired result:

    [Pretzels, Yogurt]

    Once you have identified and fixed a bug, you should remove the now unnecessary print statements as they can cause confusion later.
