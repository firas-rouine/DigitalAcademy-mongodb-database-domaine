    Fixed Arrays

    Learning Objectives:

    • Arrays in java
    • Introduction to ArrayList

    Now that you understand the primitive data types, what about more complex data structures? There have to be ways to contain more than just a single integer or string at a time. To do this we will use one of the simplest data structures: the Array.

    Fixed Arrays

    The traditional array type in Java is a fixed-size sequential collection of elements of the same type with a zero-based index. In other words, zero-based index means that accessing and setting elements is done by an integer where 0 represents the first item, and so on. To use an array, we first need to declare it, initialize and add elements or values to it. We can either do these separately or all at once.

    Declaration, initialization, and adding, in separate steps:

    int[] myArray;
myArray = new int[5];    // Initialization
myArray[0] = 4;
myArray[1] = 8;
myArray[2] = 8;
myArray[3] = 5;
myArray[4] = 9;

    Let's break this down:

    When you first declare and initialize an array, it will be an array of that size, with default values. You can also initialize the values, hard coded all at once. Notice below that in Java, to initialize a hard-coded basic array, you use curly braces instead of square brackets.

    int[] myArray = {4, 8, 8, 5, 9};

    Fixed Array Means Fixed Size

    Yep, that's right. No change in size means no adding to or removing the last element. In both these examples, the length of the array is determined by the number of elements it has. The built-in array type has a fixed size (after initialization), meaning that you cannot add or remove items from an array. In the example above, we declared that this array would contain five items. While you can re-assign those initial zeroes, you cannot decrease or increase the number of elements in this array after initialization.

    Food for thought: What would the array contain if it were declared as an array of strings like so, String[] fruits = new String[5];?

    Manipulating Fixed Arrays

    Although you cannot add extra elements or remove allocated space from an array, accessing elements to read and write values to an array should be familiar to you. Let's take the classic example of a swap.

    String[] fruits = {"banana", "pear", "papaya", "kiwi"};
// To swap the value at index 0 and index 3
String temp = fruits[0]; // assigns temp to "banana"
fruits[0] = fruits[3]; // { "kiwi", "pear", "papaya", "kiwi"}
fruits[3] = temp; // { "kiwi", "pear", "papaya", "banana"}

    To get the length of the array, in this case you would use fruits.length. 

    System.out.println(fruits.length); // 4
// Array is currently { "kiwi", "pear", "papaya", "banana" }
System.out.println(fruits[fruits.length - 1]); // "banana"

    Before you move on, go ahead and try experimenting with the array type on your own!

    Note About Printing an Array

    If you've been coding along, and you happen to have tried printing the entire array to see what was inside, you probably got something like this: [I@6bc7c054. This isn't an error, it's actually the default string representation of the array. The [I means it's an array of type int and @6bc7c054 is the address at which the array is stored. This isn't very helpful if you want to know what's inside the array. For this basic Java array type, you'll have to do a for loop to see all the elements. Fortunately, the ArrayList type, which you'll be learning next, will have more flexibility and functionality.
