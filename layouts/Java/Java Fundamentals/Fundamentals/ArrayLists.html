    ArrayLists (Dynamic Arrays)

    Learning Objectives:

    • Introduce ArrayLists in Java and how to use them
    • Introduce Generics
    • Start reading Java documentation

    The ArrayList Type (Dynamic Arrays)

    The ArrayList type is more like the arrays you're probably used to if you're more familiar with languages like JavaScript or Python. Although the basic fixed array type is very space and time efficient, it is limited, not only by the fixed size, but also in what you can do with it. In this case, the util module provides us with an ArrayList. There are some slight changes in how you interact with an ArrayList, but it is still a sequential zero-based index collection of elements. It is not, however, fixed in size. We can add as many items as we need to! To create and add items to an ArrayList you will need to do the following:

    import java.util.ArrayList;
ArrayList<Integer> myArray = new ArrayList<Integer>();

    Let's unpack this a bit.

    First, what is this <Integer> business? This is part of what we call generics.

    Generics <T>

    In the same way we declare which type a variable can hold, we need to declare the acceptable types an ArrayList can hold as well. Although it is a dynamically sized array, it still needs to know how much space in memory to allocate for each item in that particular ArrayList.

    Generics help to keep our compiler, and us, from making mistakes that would bite us during runtime. In fact, you may use a generic <T> in the future when defining your own classes or types, the same way the ArrayList class uses them. That is, to indicate a placeholder for a type to give other programmers the option to use their own specified type in conjunction with your class. But for now, we'll just be using them to indicate the data type an ArrayList will hold.

    < Objects Only >

    One potential drawback of ArrayLists is that they can only hold object types, or rather, references to objects, so youâ€™ll need to use those wrapper classes we talked about like Integer, or Double. The String class is already an object type, so ArrayLists can store them as well.

    Manipulating ArrayLists

    For the ArrayList type, instead of using bracket notation, we use what are called getters and setters. You can add, set and get elements in the following manner:

    // Add items to the array (expanding the size)
myArray.add(10); // { 10 }
myArray.add(11); // { 10, 11 }
// To read values from the array,
// use .get() passing in the index
Integer num = myArray.get(0); // 10
    
// Overwrite values in the array
// use .set() passing in the index 
// and the value you want to change it to
myArray.set(0, 9); // { 9, 11 }

    What if you would like to add elements that are of different types? Well, all objects are inherited from the Object class. Therefore, we can have our generic as an object and add different types of objects! For example: 

    import java.util.ArrayList;
ArrayList<Object> things = new ArrayList<Object>();
things.add(10);
things.add("Hello");
things.add(new ArrayList<Integer>());
things.add(12.5);
        
System.out.println(things); // { 10, "Hello", [], 12.5 }

    Check out the ArrayList documentation. The most used methods are add, clear, clone, contains, get, indexOf, isEmpty, remove, size. Try them out and see what you can do with the ArrayList class.

    Note: The <> are informally referred to as the "diamond."
