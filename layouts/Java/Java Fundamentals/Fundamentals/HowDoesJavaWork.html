    How Does Java Work?

    Learning Objectives:

    • Understand how and why Java source code is compiled and run
    • Understand the difference between compiled and interpreted languages
    • Gain a high-level understanding of the Java Development Kit
    • Be able to explain what the Java Virtual Machine does

    Source Code vs. Machine Code

    What is a computer program? Put simply it’s a set of instructions that a machine can read and execute. Machine code is written as a sequence of 1’s and 0’s. This is called binary code. It is not practical for humans to code in binary, although it was done initially. Instead, we code in higher level languages like Java, that are human readable.

    However, before our instructions can be executed by a computer, in other words, before run time, our human-written source code needs to be translated into machine code to be executed. And there are two ways this is typically done. Source code can be interpreted or compiled. If you have coded in JavaScript or Python, you have already had some exposure to largely interpreted languages. Most languages are considered either interpreted or compiled, although many use a combination of both. Java is considered a compiled language.

    Interpreters

    Interpreted languages are read and executed line by line. Imagine you are a child unable to read a recipe, and you need an adult to help you carry out each instruction. Much like an adult reading out a recipe line by line for a child, interpreters read and execute each line of code one by one.

    Benefits

    Drawbacks

    More flexibility

    Slower, less efficient

    Dynamic typing (more on that later)

    Smaller source code size, easier to write

    Platform independent

    Compilers

    On the other hand, let’s say you found a recipe written in a different language, and sent it off to be translated by a native speaker. As you can imagine, while sending it off to be translated might take some time, once you have the written translation, you can follow the directions much more quickly every time you make the recipe. Compiled languages are translated all at once before the code is executed, at compile time, and produce a build of the entire code before executing it at runtime. This has some benefits and some drawbacks.

    Benefits

    Drawbacks

    Performance; Faster, more efficient

    *Platform dependent (But not for Java!)

    Greater control over memory usage, resources

    		 More involved syntax and code considerations for developers

    A Brief Overview of the JDK

    Java, unlike other compiled languages such as C++ is platform independent. This means that compiled versions of the code can be executed directly by different operating systems like MacOS and Windows, even though they have different machine codes.

    Let’s break down the process a bit. We write our source code as plain text in a .java file. In order for Java to be compiled and run on many different platforms, it is first compiled into a bytecode file, which has the extension of .class. This is done by the dev tools included in the Java Development Kit, and will be the same bytecode for any platform, Windows, Mac, Linux etc.

    Finally, the Java Virtual Machine compiles the bytecode into the machine code of that particular operating system.
